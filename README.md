# 📦 Система учета оборудования (Asset Tracker)

Веб-интерфейс для инвентаризации и учета IT-оборудования с генерацией QR-кодов и возможностью интеграции с 1С.

## 🚀 Основные возможности
*   **Инвентарный список**: Просмотр, добавление и удаление оборудования.
*   **Живой поиск**: Мгновенная фильтрация по названию или серийному номеру.
*   **Цветовые статусы**: Визуальное разделение (На складе, У сотрудника, В ремонте).
*   **QR-генерация**: Создание QR-кодов на стороне сервера для маркировки техники.
*   **Печать этикеток**: Быстрая отправка QR-кода на принтер.
*   **Кэширование**: Оптимизация нагрузки на сервер за счет кэширования QR-кодов в браузере.

## 🛠 Технологический стек
*   **Backend**: Node.js, Express.js
*   **Frontend**: HTML5, CSS3 (Modern UI), Vanilla JavaScript
*   **База данных**: PostgreSQL
*   **Библиотеки**: `qrcode` (Node.js), `pg` (Postgres driver), `cors`

## 📋 Требования
*   Node.js (v14 или выше)
*   PostgreSQL (v12 или выше)

4. Настройка базы данных (PostgreSQL)
Создайте базу данных equipment_db и выполните SQL-запрос для создания таблицы:
sql
CREATE TABLE assets (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    serial_number VARCHAR(100) UNIQUE NOT NULL,
    status VARCHAR(50) DEFAULT 'На складе',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Индекс для быстрого поиска
CREATE INDEX idx_serial ON assets(serial_number);
Используйте код с осторожностью.
5. Описание API (Бэкенд)
Метод	Эндпоинт	Описание
GET	/api/assets	Возвращает список всего оборудования (новые сверху).
POST	/api/assets	Добавляет новое устройство в БД.
DELETE	/api/assets/:id	Удаляет устройство по его ID.
GET	/api/qrcode?text=...	Генерирует QR-код в формате Base64 из переданной строки.
6. Ключевые функции интерфейса
Живой поиск: Фильтрация таблицы происходит на стороне клиента по массиву allAssets, что обеспечивает мгновенный отклик без запросов к БД.
Кэширование QR: При нажатии на кнопку "QR", сгенерированное изображение сохраняется в объекте qrCache. Повторное нажатие на то же устройство не вызывает запрос к серверу.
Безопасная печать: Функция printQR() создает временное окно, передает туда только изображение кода и текст, после чего вызывает системный диалог печати.
Статусы: Реализована цветовая индикация (Зеленый — Склад, Синий — Сотрудник, Желтый — Ремонт).
